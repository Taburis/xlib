Convolution#

Max Pooling#

The function of the max pool layer is boosting the traing speed and preventing the over-fitting by reducing the outputs from selecting the maximum value among the inputs within a loca region as the outputs. For instance, for a size W×HW\times HW×H max pooling layer, it will slice the input into a many small region with shape W×HW\times HW×H, (padding or stride may apply). And only the maximum value of each slices will be the output from that region.

Batch Normalization#

Reference

The motivation of the batch normalization is based on the imperical conclusion that the NN training can be boosted and stablized if the activations are normalized to have a standard Gaussian distribution over the training samples. The batch normalization (BN) is targetting to practically implement this idea and preserve the representation of each layer. Since we know that normalized activation X∼N(0,1)X\sim N(0,1)X∼N(0,1) which means it will shift the mean value of the original output xxx. This can potentially contradict to what this layer is suppose to represent. So an extra linear transform is applied to XXX.

Suppose the mini-batch of samples leads to the inputs x={x1,…,xn}x=\lbrace x_1,\dots, x_n\rbracex={x1​,…,xn​} to BN, then the BN algorithm y=BN(x∣γ,β)y = BN(x|\gamma, \beta)y=BN(x∣γ,β), where γ,β\gamma,\betaγ,β are trainable parameters, is:

  1. Calculate the mean and variance:
    μ=1n∑ixi,σ2=1n∑i(xi−μ)2,\begin{aligned} \mu &= \frac{1}{n}\sum_i x_i, \\ \sigma^2 &= \frac{1}{n}\sum_i(x_i-\mu)^2, \end{aligned}μσ2​=n1​i∑​xi​,=n1​i∑​(xi​−μ)2,​
  2. Standardized the x→Xx\to Xx→X by X=(x−μ)/(σ+ϵ)X=(x-\mu)/(\sigma+\epsilon)X=(x−μ)/(σ+ϵ) where ϵ>0\epsilon>0ϵ>0 is a small number added for stability purpose.
  3. Apply the linear transformation to get the final ouput y=γX+βy=\gamma X+\betay=γX+β.

Noticed that the transform will leads to original xxx if γ,β\gamma,\betaγ,β was trained to be γ=σ\gamma=\sigmaγ=σ and β=μ\beta=\muβ=μ.

Residual Connection (ResNet)#

Reference

Learning the ground truth T(x)T(x)T(x) for input xxx is straightfoward. However, the disadvantages come from the fact that the gradient is getting weaker when backward propagatig longer layers. This weakening makes the features extracted from the first few layers are hard to be trained properly which caused the input features are smeared. To solve this problem, instead of fitting the ground truth T(x)T(x)T(x), we can try to fit the resdiual of the ground truth, R(x)=T(x)−xR(x)=T(x)-xR(x)=T(x)−x, respected to the input xxx. For doing this, a residual connection, known as shortcut connection, is introduced:

The output from layer F(x∣θi)F(x|\theta_i)F(x∣θi​) will needs to combined the shortcut to form the input for the next layer

xi=F(xi−1∣θi)+P⋅xi−1,x_i = F(x_{i-1}|\theta_i)+P\cdot x_{i-1},xi​=F(xi−1​∣θi​)+P⋅xi−1​,

where xix_ixi​ is the output from the iii-th layers and the x0x_0x0​ stands for the raw input. The linear operator PPP is used for projection purpose in the case that the channel number is different between the xix_ixi​ and xi−1x_{i-1}xi−1​ (P=1P=1P=1 if the channel numbers are the same). For practice purpose, the projection can be done by a 1×11\times 11×1 convolution. Furthermore, the normalization is included in F(xi−1∣θi)F(x_{i-1}|\theta_i)F(xi−1​∣θi​).

The Advantage of this structure is that the input features xxx are allowed to be propagated even if the residual features Ri(x)R_i(x)Ri​(x) hadn't effectively extracted in the first few layers (due to the weakening of the gradients). This can benefits in large model training and reduce the error mostly raised by the non-proper features extracted from inputs.
