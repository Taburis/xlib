Optimization

Given a system formulated by a function L(x∣θ)\mathcal{L}(\boldsymbol{x}|\boldsymbol{\theta})L(x∣θ), the optimization is to find a point θ^\boldsymbol{\hat\theta}θ^ that maximize or minimize the function L(x∣θ)\mathcal{L}(\boldsymbol{x}|\boldsymbol{\theta})L(x∣θ) over all possible x\boldsymbol{x}x.

Gradient Searching#

One of the most practical solutions is based on the fact that gradient ∇L\nabla\mathcal{L}∇L always pointed to the fast direction that increase the L\mathcal{L}L, and the gradient is 0 at the maximum or minimum points of L\mathcal{L}L. The idea of gradient searching can be illustrated by the gradient descent algorithm (searching for minimum point) as an example:

  1. Start at a random point in parameter space θ\boldsymbol{\theta}θ
  2. Update the point following the gradent
    θ→θ−r∇θL(x∣θ),\boldsymbol{\theta} \to \boldsymbol{\theta} - r\nabla_\theta\mathcal{L}(\boldsymbol{x}|\boldsymbol{\theta}),θ→θ−r∇θ​L(x∣θ),
    where rrr is a learning rate.
  3. Repeat the steps above until converge.

In real practice, despite the cost of calculating the gradient is expensive for each step, but also several points need to be aware:

  • The gradient usually comes from a grid difference. It is critical to have a good estimation on the gradient.
  • The gradient searching may leads to a saddle point where ∇θL=0\nabla_\theta \mathcal{L}=0∇θ​L=0 but it is neither locally maximum nor minimum.
  • Choosing the learning rate is subtle task. A larger learning rate may needs fewer iterations to approach the solution, but it is also easier to past the minimum point so that the searching will move around the solution back and forth.
  • Converge standard can be tricky, it depends on the precision of the model.
  • Training sample may include noise so that the gradient may not precise.

Based on the idea of gradient searching, a variety of algorithm developed targeting to solve the optimization problem in a more practical way.

Batch Gradient Descent#

To suppress the error of the gradient raising from the fluctuation from input samples, the gradient is estimated by the mean of the gradients obtained from each of the input samples. The main purpose of the BGD is to reduce the error of the gradients at a cost of calculating the gradients for every data in the training batch.

Stochastic Gradient Descent#

The stochastic gradient descent (SGD) uses a gradient calculated from a random picked sample form training batch as the estimation for ground truth gradient. The advantage of this method is that it will only calculate the gradient once per each batch (independent on the batch size), it significantly simplified the gradient computation at the cost that the error of the SGD will makes the L\mathcal{L}L fluctuation during the searching iteration.

Another SGD variantion is specific for the case that L\mathcal{L}L is a summation over all the samples:

L=∑iL(xi∣θ),\mathcal{L}=\sum_iL(\boldsymbol{x}_i | \boldsymbol{\theta}),L=i∑​L(xi​∣θ),

where xi\boldsymbol{x}_ixi​ stands for the inputs from a samples. Since the similarity, the gradient of ∇θL\nabla_\theta\mathcal{L}∇θ​L is approximated by ∇θL\nabla_\theta L∇θ​L at a randomly picked sample xi\boldsymbol{x}_ixi​.

Momentum Estimation#

The algorithm of the momentum estimation updating the parameters based on

θt+1=θt+Δθt+1,Δθt+1=αΔθt−r∇θLt,\boldsymbol{\theta}_{t+1} = \boldsymbol{\theta}_t + \Delta\boldsymbol{\theta}_{t+1},\quad \Delta\boldsymbol{\theta}_{t+1} = \alpha\Delta\boldsymbol{\theta}_t - r\nabla_\theta \mathcal{L}_t,θt+1​=θt​+Δθt+1​,Δθt+1​=αΔθt​−r∇θ​Lt​,

where Δθ\Delta\boldsymbol{\theta}Δθ is 0 at the very beginning, ∇θLt\nabla_\theta \mathcal{L}_t∇θ​Lt​ is the graident with the parameter θt\boldsymbol{\theta}_tθt​, and α\alphaα is a decay parameter in [0,1][0,1][0,1] (usually α=0.9\alpha=0.9α=0.9). The idea comes from imaging the searching as a point moving in parameter space at a velocity Δθ\Delta\boldsymbol{\theta}Δθ with a friction proportional to the velocity, with a friction parameter α\alphaα. The velocity will be exhausted by the friction and leads to a minimum position. This method can suppress the disturbs from gradient fluctuation effectively so that L\mathcal{L}L may decrease smoothly.

Adaptive Gradient Descent#

The adaptive gradient descent (AdaGD) introduced a variation learning step defined by the inverse of the length of the gradient. The algorithm is similar to SGD except moving parameters as

θt+1=θt−rSt+1+ϵ∇θLt,St+1=St+∥∇θLt∥2.\boldsymbol{\theta}_{t+1}=\boldsymbol{\theta}_t-\frac{r}{\sqrt{S_{t+1}}+\epsilon}\nabla_\theta\mathcal{L}_t,\quad S_{t+1}=S_t+\Vert\nabla_\theta\mathcal{L}_t\Vert^2.θt+1​=θt​−St+1​​+ϵr​∇θ​Lt​,St+1​=St​+∥∇θ​Lt​∥2.

The gradient will be shrinked by a factor of 1/St1/\sqrt{S_t}1/St​​ at ttt-th step. The StS_tSt​ increase monotonically which leads to the learning step are decreasing as the iteration number increase. The ϵ\epsilonϵ is a small value to avoid singularity.

Root Mean Square Propagation#

The Root Mean Square Propagation (RMSProp) is a similar algorithm like AdaGD by varying the learning step according to the length of the gradient. Unlike the AdaGD which has a monotonic deceasing learning step, RMSProp adjust the learning rate as:

θt+1=θt−rvt+1+ϵ∇θLt,vt+1=γvt+(1−γ)∥∇θLt∥2,\boldsymbol{\theta}_{t+1}=\boldsymbol{\theta}_t-\frac{r}{\sqrt{v_{t+1}}+\epsilon}\nabla_\theta\mathcal{L}_t,\quad v_{t+1}=\gamma v_t+(1-\gamma)\Vert\nabla_\theta\mathcal{L}_t\Vert^2,θt+1​=θt​−vt+1​​+ϵr​∇θ​Lt​,vt+1​=γvt​+(1−γ)∥∇θ​Lt​∥2,

where γ∈[0,1]\gamma\in [0, 1]γ∈[0,1] is called "forgetting factor" as the history sum of gradient length will decay by a factor of γ\gammaγ for each iteration. The small ϵ\epsilonϵ constant is added here to avoid singularity.

Estimation-Maximization Algorithm#

The Estimation-Maximization Algorithm (EM-Algorithm) is often used for maximizing the likelihood function with unknown observations. To state the idea of this algorithm, we define the following elements:

  • XXX, a observable random variable with PDF fX(x)f_X(x)fX​(x) leads to the samples {Xn}\lbrace X_n \rbrace{Xn​}.
  • ZZZ, a hidden random variable (latent variable) with PDF fZ(z)f_Z(z)fZ​(z) and X⊥ZX\perp ZX⊥Z.
  • L(θ∣X)L(\theta|X)L(θ∣X) is the observed likelihood function that XXX is partially known, and the L(θ∣X,Z)L(\theta|X,Z)L(θ∣X,Z) is the complete likelihood function that X,ZX,ZX,Z are known.

The goal is find the parameter θ\thetaθ that maximizing the observed likelihood function. The EM algorithm is a iterative algorithm and let θ^(m)\hat \theta^(m)θ^(m) is the mmm-th step estimator for θ\thetaθ. The algorithm is

  1. E-step: Compute the function
    Q(θ∣θ^(m),x)=∫log⁡L(θ∣x,z)k(z∣θ^(m),x)dzQ(\theta|\hat\theta^{(m)},\boldsymbol{x})=\int\log L(\theta|\boldsymbol{x},\boldsymbol{z})k(\boldsymbol{z}|\hat\theta^{(m)},\boldsymbol{x})d\boldsymbol{z}Q(θ∣θ^(m),x)=∫logL(θ∣x,z)k(z∣θ^(m),x)dz
    where k(z∣θ,x)k(\boldsymbol{z}|\theta,\boldsymbol{x})k(z∣θ,x) is defined as
    k(z∣θ,x)=L(θ∣X,Z)L(θ∣X).k(\boldsymbol{z}|\theta,\boldsymbol{x})=\frac{L(\theta|X,Z)}{L(\theta|X)}.k(z∣θ,x)=L(θ∣X)L(θ∣X,Z)​.
  2. M-step: Computing the estimator for m+1m+1m+1 step:
    θ^(m+1)=arg max⁡θ∈ΩQ(θ∣θ(m),x).\hat\theta^{(m+1)}=\argmax_{\theta\in\Omega}Q(\theta|\theta^{(m)},\boldsymbol{x}).θ^(m+1)=θ∈Ωargmax​Q(θ∣θ(m),x).
