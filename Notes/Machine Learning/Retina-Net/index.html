Retina Net

It is a generalized Fast Region-based Convolutional Network (Fast R-CNN) architecture improved the prediction precision by adapting feature pyramid for feature augmentation. Two ouput heads mount on the feature pyramid net (FPN) modified from other CNNs (like residual-50 or VGG-16). The schematic view of the RetinaNet architecture showing below:

Feature Pyramid Net (FPN)#

The FPN is used as backbone to provide features in variaty of size. It is expected that the coarser features should have better semantically information while the signal with higher resolution features should have better spatial information. In the Retina Net architecture, the output layers from the ResNet {C2,C3,C4,C5}\lbrace C_2, C_3, C_4, C_5\rbrace{C2​,C3​,C4​,C5​} is used to form the FPN and the outputs are denoted as {P2,P3,P4,P5}\lbrace P_2,P_3,P_4,P_5\rbrace{P2​,P3​,P4​,P5​}. The convolution over these FPN will servese like sliding windows over the original images as illustrated below:

Anchor Boxes#

The anchor box used in the RetinaNet is similar to the setup in the Faster R-CNN, an improved version of Fast R-CNN. The anchor boxes are the bounding boxes anchored on each sliding window location, as illustrated below:

A bounding box with a scale and aspect ratio is anchored at the center of the sliding window. Three aspect ratios, (2:1, 1:1, 1:2), and three scales, (1, 0.5, 0.25), are used so that there are 9 bounding boxes anchored for each sliding windows.

Heads#

The features from FPN is used as input for two parallel subnets to make classification and box regression separately.

The Classification Subnet#

A simple fully convolution net (FCN) is used to taking the features as inputs to classify the objects for each anchors. It has 4 hidden layers with 3×33\times33×3 kernel, 256 channels and the ReLU is used as activations, and The output layer is a 3×33\times 33×3 convolution layer with K×AK\times AK×A outputs where KKK is number of object types and AAA is the anchor size (how many anchor boxes for each sliding window). The output shape should be W×H×K×AW\times H\times K\times AW×H×K×A where W×HW\times HW×H is the spatial coordinates of features and each spatial location is assigned anchor with size AAA and each anchor is assigned a vector with length KKK to represent the object type.

The Regression Subnet#

The architecture of the regression subnet is the same as the classification subnet except the output layer is 4×A4\times A4×A channels represented the coordinate of the bounding boxes. The output shape is the W×H×A×4W\times H\times A\times 4W×H×A×4, each of W×HW\times HW×H represents an anchor box with AAA aspect ratios or scales, 4 digits are used to represents the coordinate of the box.

The box regression targets are generated by assigning the ground truth boxes to the anchor boxes by intersection over union (IOU) defined as

IoU=A∩A0A∪A0,\text{IoU} = \frac{A\cap A_0}{A\cup A_0},IoU=A∪A0​A∩A0​​,

where AAA stands for the predicted area and A0A_0A0​ is the truth area. The matching criterion is that two boxes are matched to each other if the IOU of them is larger than 0.5. Those matching IOU values between 0.4 to 0.5 are ignored (not used in the paper). The matching IOU below 0.4 is considered as unmatched. The higher the IOU, the better matching of two boxes. The largest IOU matching ground truth is assigned to the anchor boxes accordingly, both the class type and box coordinates.
