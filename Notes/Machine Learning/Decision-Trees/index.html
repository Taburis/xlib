Decision Trees

The decision tree algorithms are supervised learning for solving regression and classification problems. The problems can be formulated as follows: Given datasets containing labels {yn}\lbrace y_n\rbrace{yn​} and features {xn}\lbrace x_n\rbrace{xn​} paired as (xi,yi)(x_i,y_i)(xi​,yi​), we need to construct a function or machine y=f(x)y=f(x)y=f(x) to predict the label yyy based on the input features xxx. The label could be discrete (classification) or continuous (regression) case.

Decision Tree Algorithm#

The Classification and Regression Tree (CART) is a term referring general decision tree algorithms. In CART algorithms, the machine f(x)f(x)f(x) is implemented as a tree. Mostly a binary tree is used for simplicity (ideally, any tree can be split into a binary tree). To build a CART, a metric to measure the performance of the model is needed. This metric is a functional evaluated on the samples using the iii-th feature. With it, an information gain at a split point on the feature can be defined accordingly. Furthermore, one can search the split point to maximize the information gain, and build a child node if necessary.

Given FFF features {XF}\lbrace X_F\rbrace{XF​}, the feature boundary or decision boundary BBB is the direct product of FFF intervals

B=[a1,b1]⊗⋯⊗[aF,bF].B=[a_1,b_1]\otimes\dots \otimes[a_F,b_F].B=[a1​,b1​]⊗⋯⊗[aF​,bF​].

Split the iii-th feature from the boundary BBB at ppp will lead to two sub boundaries:

BL=[a1,b1]⊗⋯⊗[ai,p]⊗⋯⊗[aF,bF]BR=[a1,b1]⊗⋯⊗[p,bi]⊗⋯⊗[aF,bF].\begin{aligned} B_L&=[a_1,b_1]\otimes\dots \otimes[a_{i},p]\otimes\cdots\otimes[a_F,b_F]\\ B_R&=[a_1,b_1]\otimes\dots \otimes[p, b_i]\otimes\cdots\otimes[a_F,b_F]. \end{aligned}BL​BR​​=[a1​,b1​]⊗⋯⊗[ai​,p]⊗⋯⊗[aF​,bF​]=[a1​,b1​]⊗⋯⊗[p,bi​]⊗⋯⊗[aF​,bF​].​

The idea of the decision tree learning is to partition the features into small boundaries iteratively until the partition is fine enough for prediction. The iterations built a CART can be illustrated as the following:

Classification Trees#

Suppose the FFF features {XF}\lbrace X_F\rbrace{XF​} assigned for each data labeled by yyy where CCC possible classes in total, several metrics can be used for classification problem. Given a feature boundary BBB, these metrics can be:

  • Entropy is a metric used for discrete features. The entropy Hi(SB)H_i(S_{B})Hi​(SB​) with respect to the iii-th feature component, where SBS_{B}SB​ is the samples falling in the feature boundary BBB, is defined as

    where pk(x)p_k(x)pk​(x) is the probability to find a kkk-th class data satisfying Xi=xX_i=xXi​=x in the sample SBS_{B}SB​. In general, smaller entropy means less randomness and better information. Hi(SB)=0H_i(S_{B})=0Hi​(SB​)=0 means a perfect classification by using the iii-th feature. The entropy will calculated for all features and the feature with the smallest entropy is picked for splitting. Given a split that B→BL,BRB\to B_L, B_RB→BL​,BR​, the information gain G(B,BL,BR)G(B, B_L, B_R)G(B,BL​,BR​) is defined as
    GH(B,BL,BR)=H(B)−α⋅H(BL)−(1−α)⋅H(BR),G_H(B, B_{L}, B_{R}) = H(B)-\alpha\cdot H(B_L)-(1-\alpha)\cdot H(B_R),GH​(B,BL​,BR​)=H(B)−α⋅H(BL​)−(1−α)⋅H(BR​),
    where α\alphaα is the fraction of the data from BBB falls into BLB_LBL​. The split point can be determined from searching the maximum information gain from the possible split points along the XiX_iXi​.
  • Gini Impurity IG(B)I_G(B)IG​(B) is defined as
    IG(B)=∑i=1C∑x∈Bipk(x)(1−pk(x))=1−∑k=1Cpk2(x),I_G(B) = \sum_{i=1}^C\sum_{x\in B_i}p_k(x)(1-p_k(x))=1-\sum_{k=1}^Cp^2_k(x),IG​(B)=i=1∑C​x∈Bi​∑​pk​(x)(1−pk​(x))=1−k=1∑C​pk2​(x),
    where pk(x)p_k(x)pk​(x) is the probability of finding a kkk-th class data with Xi=xX_i=xXi​=x from the sample in BBB. A perfect classification, only one class in the boundary, leads to 0 impurity, so it is smaller the better. Suppose a split that divide the boundary into two subsets BL,BRB_L, B_RBL​,BR​, and let α\alphaα and 1−α1-\alpha1−α stands for the fraction of original samples fall into the two subsets.     IG(BL,BR)=α⋅IG(BL)+(1−α)⋅IG(BR).I_G(B_L,B_R)= \alpha\cdot I_G(B_L)+(1-\alpha)\cdot I_G(B_R).IG​(BL​,BR​)=α⋅IG​(BL​)+(1−α)⋅IG​(BR​).
    The impurity gain G(BL,BR)G(B_L,B_R)G(BL​,BR​) is defined as the difference:
    GI(B,BL,BR)=IG(B)−IG(BL,BR).G_I(B,B_L,B_R)=I_G(B)-I_G(B_L,B_R).GI​(B,BL​,BR​)=IG​(B)−IG​(BL​,BR​).
    Again, the split point is searched through the XiX_iXi​ to find the point maximum the gain GIG_IGI​.
Impurity vs Entropy

Practicing experience shows that entropy method is slightly better performed than the impurity. However, the impurity metric is faster than the entropy calculation due to the relative simple expression.

Extend to Continuous features

To apply these metrics for continuous features, the feature ranges are needed to be quantized into subintervals, then the summation over the discrete feature values in the metrics above, the features are summed over the subintervals for the continuous features. The intervals are usually divided into two subintervals so that a binary search can be used to find the maximum split point.

Regression Trees#

For regression problem, the input features are supposed to be continuous. For the regression tree, a linear approximation, mostly the constant approximation, is driven by using the data in the feature boundary BBB, so that a more fine partition will leads to a better approximation. Let the y^j(x)\hat y_j(x)y^​j​(x) be the jjj-th of the output from the regression tree, then y^j(x)=CBx\hat y_j(x)=C_{B_x}y^​j​(x)=CBx​​ where CBxC_{B_x}CBx​​ is a constant and BxB_xBx​ is the finest feature boundary contained the xxx. Pruning#

Pruning is a term referring to remove subtrees or leaves from a trained decision tree to advance the speed and simplicity. To carry out the pruning, a complexity measure for a tree is needed. It run through all the nodes in a either top-down or bottom up order, removing the subtrees if the complexity reduction is worth comparing to the accuracy lost.
