Tree Traversal

Binary Tree Traversal#

Binary tree is a tree where have at most two branches (left or right) at each node. The node of a binary tree is usually denoted as (S,L,R)(S, L, R)(S,L,R) where SSS is the node itself and L,RL,RL,R is the left or right side descendants. The level (or depth) nnn of each node is nnn-th descendant of the root node.

There are generally 3 types of order of traversal through a binary tree:

  1. Pre-order visit the nodes in a order that node-> left-> right
  2. In-order: left->node->right
  3. Post-order: left->right->node

These three types of order of traversal can be implemented iterative in a very similar way. Suppose action is the function we'd like to impose on each node, then the traversal pesudo-code is:

def iter(node, action):    if not node: return    action(node) # pre-order     iter(node.left)    action(node) # in-order     iter(node.right)    action(node) # post-order

The pre-order is a special case of DFS. A BFS is a level-order traversal since it will visit all nodes in the same level before it move to the next level.

Depth-first Search (DFS)#

A similar algorithm like BFS, with the searching priority focused on the depth of the tree/graph G=(V,E)G=(V,E)G=(V,E). A iterative pesudo-code of DFS is:

def DFS(root, target):    if root == target: return root    root.discovered = True    for node in root.neighbours:        if node.discovered: continue        DFS(node, target)

Also a non-iterative implementation is similar to the BFS code:
